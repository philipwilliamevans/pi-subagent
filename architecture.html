Pi Subagent Architecture

A one-page technical map of how the extension discovers specialist agents, teaches the parent Pi agent about them, spawns isolated child Pi processes, handles sessions, and reports results.

Execution Flow

Foreground

subagent

Runs one or more specialist calls and waits for all results. Calls are validated, session identities are derived, locks are acquired, child processes are spawned, JSON events are streamed, and a concise parent-facing summary is returned.

  • Up to 8 calls per invocation.
  • Up to 4 child processes at once.
  • Supports named persistent sessions.
  • Supports initialContext: "parent" when a parent snapshot is available.
Background

subagent_start

Creates an in-memory job, starts child processes asynchronously, and returns a job ID. Completion can inject a message and optionally trigger a parent turn.

  • Root parent session only.
  • No persistent sessions.
  • No parent-context snapshots yet.
  • Use subagent_status, subagent_cancel, and subagent_result after start.

Sessions and Context

Empty context

The default. The child starts with --no-session unless a named session is supplied.

Parent context

The parent session header and branch are serialized as JSONL and passed to the child. The snapshot cwd is rewritten to the child cwd.

Named session

A deterministic child session ID is derived from parent session ID, cwd, agent name, and logical handle.

Named sessions require a persisted parent Pi session. The same handle can intentionally map to different sessions when the agent or cwd changes.

Background Job Lifecycle

States

Jobs move through running, cancelling, cancelled, completed, or failed. Each call also tracks a finer phase such as queued, spawning, running, completed, failed, or cancelled.

Result delivery

onComplete defaults to trigger, which injects a completion message and starts a parent turn. message injects only. silent records results in memory only.

Risk Areas

Non-durable background jobs

Jobs are stored in memory. Restarting the parent extension process loses job status and results.

Shared working tree

Background children can edit the same files as the parent or each other. The extension warns but does not serialize file writes.

Advisory session locks

Locks are filesystem directories with heartbeat metadata. Stale recovery is manual.

Pi JSON event coupling

Success semantics depend on recognized event types: message_end, turn_end, and agent_end.
